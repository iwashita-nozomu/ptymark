# ptymark v0.1.0-alpha.2

This supportability release makes the existing PTY/ConPTY renderer diagnosable and bounded when an external renderer misbehaves.

## Highlights

- `ptymark doctor`, deterministic `ptymark.doctor.v1` JSON, and atomic redacted support-report files.
- Stable `ready`, `degraded`, and `unusable` states with exit codes `0`, `10`, and `20`.
- Public-safe diagnostic finding codes for configuration, installation state, host, terminal, engine, browser, presenter, timeout, output-limit, process-exit, fallback, and session modes.
- A ten-second monotonic hard deadline for external render/presentation attempts.
- An eight MiB renderer/artifact/display bound and a one MiB bound on later terminal output held behind an unresolved semantic block.
- Exact-source recovery in non-strict mode, original output ordering, process-tree cleanup, and no cache admission for failed/timed-out/cancelled/invalid attempts.
- GitHub issue forms and packaged troubleshooting guidance aligned with the doctor-v1 redaction contract.

## Privacy and safety

Default doctor and support-report output excludes semantic source, child environment, credentials, raw source-bearing renderer stderr, sensitive path prefixes, and terminal-control bytes. Default doctor performs no installation, download, network access, renderer/browser execution, child launch, or mutation.

Source, safe, and private modes retain their alpha.1 behavior. Renderer recovery never terminates the user's PTY/ConPTY child.

## Compatibility and known limitations

- This remains an unsigned alpha prerelease distributed as Linux, macOS, and Windows archives.
- Guided first-run rendering and generated WezTerm setup remain follow-up work.
- CJK/grapheme-width, screen-reader, and broader accessibility completion remain follow-up work.
- Upgrade, rollback, uninstall, purge, signing, package-manager channels, persistent workers, disk cache, and terminal image protocols are not included.
